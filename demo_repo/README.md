# Payment Service API

A production-ready Flask-based payment processing service with user authentication, payment processing, and refund capabilities.

## Features

- **User Authentication**: JWT-based authentication system
- **Payment Processing**: Secure payment charge processing with validation
- **Refund Management**: Process refunds for completed payments
- **Payment History**: Retrieve payment records and user payment history
- **RESTful API**: Clean REST endpoints with proper error handling

## Tech Stack

- **Framework**: Flask 2.3.3
- **Authentication**: Flask-JWT-Extended
- **Database**: SQLAlchemy ORM (SQLite for development, PostgreSQL recommended for production)
- **Testing**: Pytest

## Installation

```bash
# Install dependencies
pip install -r requirements.txt

# Run the application
python app.py
```

The server will start on `http://localhost:5000`

## Database Models

### User
- `id`: Unique identifier
- `email`: User email (unique)
- `tier`: Subscription tier (`free` or `premium`)
- `created_at`: Account creation timestamp

### Payment
- `id`: Unique identifier
- `user_id`: Foreign key to User
- `amount`: Payment amount in USD
- `status`: Payment status (`pending`, `completed`, `failed`, `refunded`)
- `card_token`: Tokenized card information
- `created_at`: Transaction timestamp
- `updated_at`: Last update timestamp

## API Endpoints

### Authentication

#### POST /auth/login
Authenticate user and receive JWT token.

**Request:**
```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

**Response:**
```json
{
  "access_token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "tier": "free",
    "created_at": "2024-01-01T12:00:00"
  }
}
```

### Payments

#### POST /payments/charge
Process a payment charge. Requires authentication.

**Headers:**
```
Authorization: Bearer <access_token>
```

**Request:**
```json
{
  "amount": 99.99,
  "card_token": "tok_visa_4242424242424242"
}
```

**Response:**
```json
{
  "payment_id": 123,
  "user_id": 1,
  "amount": 99.99,
  "status": "completed",
  "created_at": "2024-01-01T12:00:00"
}
```

#### GET /payments/{payment_id}
Retrieve a specific payment. Requires authentication.

**Headers:**
```
Authorization: Bearer <access_token>
```

**Response:**
```json
{
  "id": 123,
  "user_id": 1,
  "amount": 99.99,
  "status": "completed",
  "created_at": "2024-01-01T12:00:00",
  "updated_at": "2024-01-01T12:00:00"
}
```

#### POST /payments/refund
Process a refund for a completed payment. Requires authentication.

**Headers:**
```
Authorization: Bearer <access_token>
```

**Request:**
```json
{
  "payment_id": 123
}
```

**Response:**
```json
{
  "payment_id": 123,
  "amount": 99.99,
  "status": "refunded",
  "refunded_at": "2024-01-01T12:30:00"
}
```

#### GET /users/{user_id}/payments
List all payments for a user. Requires authentication.

**Headers:**
```
Authorization: Bearer <access_token>
```

**Query Parameters:**
- `limit` (optional): Maximum number of payments to return (default: 50)

**Response:**
```json
{
  "payments": [
    {
      "id": 123,
      "user_id": 1,
      "amount": 99.99,
      "status": "completed",
      "created_at": "2024-01-01T12:00:00",
      "updated_at": "2024-01-01T12:00:00"
    }
  ],
  "count": 1
}
```

## Error Handling

The API returns appropriate HTTP status codes:

- `200 OK`: Successful GET request
- `201 Created`: Successful POST request (resource created)
- `400 Bad Request`: Invalid request data
- `401 Unauthorized`: Missing or invalid authentication
- `403 Forbidden`: Insufficient permissions
- `404 Not Found`: Resource not found
- `500 Internal Server Error`: Server error

Error responses include a descriptive message:
```json
{
  "error": "Payment amount must be greater than 0"
}
```

- `SECRET_KEY`: Flask secret key (default: dev-secret-key-change-in-production)

- `DATABASE_URL`: Database connection string (default: sqlite:///payments.db)

## Security Considerations

- All payment endpoints require JWT authentication
- Users can only access their own payment data
- Card tokens should be generated by a payment gateway (Stripe, PayPal, etc.)
- In production, use HTTPS for all API calls
- Store secrets in environment variables, not in code
- Use a production-grade database (PostgreSQL, MySQL)

## Business Logic

### Payment Validation
- Amount must be greater than 0
- Amount cannot exceed $10,000
- Card token must be at least 10 characters

### Refund Rules
- Only completed payments can be refunded
- Payments cannot be refunded twice
- Refunds are processed immediately
